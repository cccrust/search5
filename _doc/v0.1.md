# v0.1 規劃：搜尋引擎系統

## 1. 專案概述

名稱：search5
類型：網頁搜尋引擎（Web Search Engine）
主要功能：對本地 HTML 檔案建立索引並搜尋

## 2. 技術選型

- 語言：Rust
- Web Framework：Axum
- 搜尋引擎：nanofts（使用 n-gram 處理中文，無需分詞）
- HTML 解析：scraper
- 并发处理：tokio

## 3. 架構設計

```
src/
├── main.rs           # 入口點
├── indexer/          # 索引模組
│   ├── mod.rs
│   └── indexer.rs    # 建立索引 (nanofts)
├── search/           # 搜尋模組
│   ├── mod.rs
│   └── engine.rs     # 搜尋引擎
├── parser/           # 解析模組
│   ├── mod.rs
│   └── html.rs       # HTML 解析
└── api/              # API 模組
    ├── mod.rs
    └── routes.rs     # REST API
```

## 4. 功能列表

### v0.1 完成事項

1. **HTML 解析**
   - 讀取 data/ 目錄下的 HTML 檔案
   - 解析標題與內文
   - 提取文字內容

2. **索引系統**
   - 使用 nanofts 建立反向索引（Inverted Index）
   - 使用 n-gram 處理中文（無需分詞）
   - 支援任意字元組合匹配
   - 儲存索引至硬碟

3. **搜尋引擎**
   - 關鍵字匹配
   - BM25 排序演算法
   - 支援中文搜尋

4. **REST API**
   - `POST /index` - 建立索引
   - `GET /search?q=關鍵字` - 搜尋
   - `GET /index/status` - 索引狀態

## 5. API 設計

### 建立索引 API
```
POST /index
Body: { "data_dir": "data/" }
Response: { "indexed": 20, "documents": 20 }
```

### 搜尋 API
```
GET /search?q=關鍵字
Response: {
  "results": [
    { "url": "data/01_programming.html", "title": "程式設計基礎", "snippet": "...", "score": 1.5 }
  ]
}
```

### 狀態 API
```
GET /index/status
Response: { "documents": 1000, "words": 5000 }
```
