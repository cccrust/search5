# v1.0-sql5 規劃文件：將 nanofts 置換為 sql5

## 目標

將 search5 的全文檢索引擎從 nanofts 替換為 sql5，利用 sql5 的 FTS5 支援來實現相同功能。

## 為什麼選擇 sql5

- sql5 是作者開發的 SQLite 相容資料庫
- 內建 CJK FTS5 全文檢索（bigram tokenization）
- 支援 AND/OR/NOT 查詢語法
- 可作為 Rust 庫直接使用

## 主要差異

| 面向 | nanofts (目前) | sql5 (目標) |
|------|----------------|-------------|
| 類型 | 專門的 FTS 引擎 | 完整 SQL 資料庫 |
| 持久化 | 自有格式 (.nfts) | SQLite 資料庫 (.db) |
| API | UnifiedEngine | FtsTable 或 SQL 語句 |
| 搜尋語法 | nanofts 語法 | FTS5 MATCH 語法 |

## API 對照

### nanofts 使用方式
```rust
use nanofts::{EngineConfig, UnifiedEngine};

let config = EngineConfig::persistent(path);
let engine = UnifiedEngine::new(config).expect("Failed to create engine");

// 建立索引
let mut fields = HashMap::new();
fields.insert("url".to_string(), doc.url.clone());
fields.insert("title".to_string(), doc.title.clone());
fields.insert("content".to_string(), doc.content.clone());
engine.add_document(id, fields).ok();

// 搜尋
let result = engine.search(query)?;
let doc_ids = result.top(limit);
```

### sql5 使用方式

#### 方式一：直接使用 FtsTable（記憶體）
```rust
use sql5::fts::FtsTable;

let mut table = FtsTable::new("documents", vec!["url".into(), "title".into(), "content".into()]);

// 建立索引
table.insert(vec![doc.url, doc.title, doc.content]);

// 搜尋
let results = table.search("rust AND programming");
// results: Vec<(rowid, score, column_values)>
```

#### 方式二：使用 DiskStorage（持久化）
```rust
use sql5::pager::storage::DiskStorage;
use sql5::catalog::Catalog;

// 開啟或建立資料庫檔案
let storage = DiskStorage::open("search5.db").expect("無法開啟資料庫");
let mut catalog = Catalog::new(storage);

// 建立 FTS 表
// ... 使用 SQL 或直接操作
```
